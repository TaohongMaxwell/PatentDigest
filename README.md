# 📄 PatentDigest ⚡

**通用专利文档智能解析 Skill & Dify 工作流**  

## 🤔 技术背景

专利文件具有高度一致的格式要求和明确的结构标识（如内容顺序、权利要求书/说明书格式、段落序号等），大幅降低了 LLM 训练、微调及运用的技术门槛。因此，「自动化处理专利文件」是 LLM 最容易落地的场景之一。

## ⚡ 实施效果

本工具能够自动提取专利文件关键信息，生成结构化摘要与权利要求分析，帮助用户快速掌握技术方案。输出内容包括：
- **基本信息**：名称、申请号、申请日等。
- **摘要精炼**：300字以内的关键摘要。
- **权利要求分析**：结构化的权利要求总结。
- **创新点与技术方案**：提炼现有技术不足、本发明解决的问题及有益效果。

## ✨ 核心功能

- 📌 **多格式支持**：兼容 PDF/DOCX/TXT 等格式的专利文档（需先转为文本）。
- 🧠 **模块化分析**：采用分步 Prompt 链，确保信息提取的准确性。
- 📊 **标准化输出**：生成统一的 Markdown 格式报告。
- 🔧 **平台无关**：主推的 `Skill` 模式不依赖任何特定 AI 平台，可直接在 OpenClaw、OpenCode、Hermes 等通用 Agent 中运行。
- 🔄 **多专利对比**：支持对多个同领域专利进行独立解析后的横向对比分析。

## 🛠️ 基本原理与输入输出

### 基本原理
PatentDigest 模拟了专业专利工程师的阅读习惯。它首先将长文档拆解为“基本信息”、“摘要”、“权利要求”和“说明书”四个模块并行提取；随后对提取出的内容进行深度总结和关键词过滤；最后将所有精华内容合并成一份标准化的 Markdown 报告。为了节省资源，它在处理过程中会自动将中间结果存入临时文件夹。

### 输入信息
- **原始文件**：支持 `.pdf`, `.docx`, `.txt` 等格式的专利全文。
- **执行指令**：用户可指定是否需要开启“多专利对比模式”。

### 输出信息
- **结构化报告 (`report.md`)**：包含发明名称、申请信息、精炼摘要、权利要求总结、五大创新维度分析及核心技术方案。

## ⚙️ 前置配置建议

- **OCR 技能联动**：由于大量早期专利为扫描版图片 PDF，建议在环境中预装 OCR 技能（如 `pdf-document-handler`）。当 Agent 检测到文档无法直接提取文字时，会自动引导你启用 OCR 功能。
- **模型选择**：推荐使用 Qwen2.5-7B-Instruct 及以上版本，或 DeepSeek-R1 等逻辑推理较强的模型。
